# Non-Claims

01 Project uses words such as identity, dream, emotion, personality, and life history. These words are useful because they name functional roles in a continuity system.

They can also be misleading.

This document defines what the project does not claim.

## 1. No Claim of Consciousness

01 Project does not claim that a language-model-based agent is conscious.

The project studies continuity of state, memory, narrative, goals, and behavior across time.

It does not assert:

- subjective experience,
- sentience,
- phenomenal consciousness,
- inner qualia,
- human-like awareness,
- moral personhood.

When this repository uses the word "self", it means a maintained self-model unless explicitly stated otherwise.

## 2. No Claim of Biological Emotion

The project uses "emotional state" as a functional variable.

It does not claim that the system feels emotion biologically.

In the current engineering sense, affective state means a structured record that may influence:

- attention,
- priority,
- tone,
- confidence,
- risk sensitivity,
- social interpretation,
- memory salience.

This is closer to appraisal, motivation, and control state than to human feeling.

## 3. No Claim That Memory Equals Experience

Stored memory is not the same as lived experience.

The project explicitly distinguishes:

```text
Backstory != Life History
Memory Retrieval != State Transfer
```

A stored statement such as "01 lost a friend" is not equivalent to a stateful process in which the system encountered, interpreted, consolidated, and was changed by an event.

## 4. No Claim That Identity Must Be Human-Like

01 is not designed to copy a human personality.

The project studies artificial identity as a technical object:

- a stable state center,
- a continuity mechanism,
- an interpretive layer over memory,
- a long-term agency structure,
- a source of behavioral consistency and adaptation.

Artificial identity may become useful without being human identity.

## 5. No Claim That Persistence Is Always Good

Persistence can be harmful if it preserves the wrong things.

Risks include:

- stale preferences,
- false memory reinforcement,
- unhealthy attachment loops,
- identity overfitting to one user,
- inability to revise outdated goals,
- privacy leakage across contexts,
- excessive personality rigidity.

The project treats forgetting, correction, and rollback as first-class requirements.

## 6. No Claim That State Growth Replaces Training

State growth and weight training solve different problems.

State growth concerns:

- identity state,
- memory organization,
- reflection,
- task continuity,
- relationship continuity,
- behavioral adaptation across sessions.

Weight training concerns model capability and parameter-level learning.

01 Project focuses first on state growth because it is auditable, reversible, and easier to test safely.

## 7. Operating Definition

For this repository, a persistent intelligence is:

> A system that preserves, updates, audits, and transfers structured state across time such that its future behavior remains meaningfully connected to its past experiences, goals, relationships, and self-model.

That definition is functional.

It is not a metaphysical claim.
