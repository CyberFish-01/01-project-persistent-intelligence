# Foundation Status

Chinese version: [FOUNDATION_STATUS_ZH.md](./FOUNDATION_STATUS_ZH.md)

P53 records the current foundation status after P51. This document is a planning
and maintenance artifact, not a runtime specification.

## What Exists

- Local 01 Core runtime with `StateStore`.
- Identity Core with high-gate protection.
- Memory stores with provenance, lifecycle, import, archive, and review paths.
- Dream Engine with candidate proposals and Dream artifacts.
- Claim Graph with evidence, conflict, and review-oriented repair.
- Task Hub with active tasks, procedural memory, reflection, cautionary memory,
  and review queues.
- Context Builder with policy, attribution, governance signals, and coverage
  review.
- Append-only Event Log with replay checks, projection reports, rollback
  preview, retention review, and replayability assessment.
- Reconstruction readiness reports for payload/diff coverage, evidence schema,
  coverage mapping, gap prioritization, checklist review, and evidence request
  lifecycle.
- Stateful Memory and Growth Semantics reports.
- Growth Candidate Review as a review-only governance object.
- Deterministic local evaluation scenarios and baselines.

## What Is Missing

- Object-level reconstruction reducer contract.
- Payload/diff capture mechanism.
- Recall event write policy.
- Growth candidate lifecycle.
- Temporal Awareness runtime.
- Minimal encoding policy for stateful memory.
- Subject Kernel / World Seed boundary.
- Exploration / Serendipity Engine.
- Long-duration endurance evaluation.

## Stable Foundation

- 01 Core owns state.
- Platforms and adapters translate; they do not own identity.
- Continuity is state transfer, not memory retrieval.
- Dream proposes; review decides.
- Identity Core is protected by high-gate review.
- Event Log is append-only audit evidence.
- Review-only reports do not mutate state.
- Growth candidate is not growth.

## Concepts To Push Back

- Temporal Awareness implementation.
- Recall event writes.
- Growth lifecycle.
- Memory rewrite.
- Automatic identity mutation.
- Relationship memory implementation.
- Companion/social layer.
- UI, AstrBot specialization, and adapter expansion.
- Reconstruction reducer execution.
- Event compaction.

## Current Foundation State

The project has a strong review/audit foundation and a clear continuity thesis.
It is ready for consolidation, schema review, and careful reconstruction design.
It is not ready for automatic growth, temporal runtime, companion product work,
or event compaction.

The next engineering loops should prefer reducing overlap and clarifying
contracts before adding new runtime behavior.
