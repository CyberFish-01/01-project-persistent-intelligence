## 创始人快照 / Founder Snapshot

01 Core today is a continuity foundation for persistent intelligence. It is not
a product, companion, chatbot skin, autonomous tool runner, or identity-changing
growth engine.

What exists now:

- a local prototype with durable state references such as `StateStore`, event
  records, memory import, Dream consolidation, HTTP/API references, and adapter
  references;
- a much larger foundation layer covering identity, state transfer, event
  sourcing, replay, reconstruction readiness, stateful memory, growth review,
  temporal concepts, capability boundaries, and visual naming;
- explicit blocked boundaries around identity mutation, memory rewrite, recall
  writes, growth execution, temporal runtime, CTM runtime, tool autonomy,
  companion behavior, product UI, adapters, reducers, and event compaction.

The most important current state:

```text
01 Core has a protected foundation and a visible boundary map.
It is not yet an application layer.
The next useful step is choosing what to observe or plan, not adding power.
```

The observatory layer is needed because the foundation now has many concepts and
reports. A founder should not need to read every RFC to know what is implemented,
what is report-only, what is RFC-only, what is future direction, and what must
remain blocked.
